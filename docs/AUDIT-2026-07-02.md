## 1. High impact — 3D stack loads on first paint

`dist/assets/vendor-drei-*.js` is **1.12 MB (309 KB gzip)** and is both
`modulepreload`ed from `index.html` and statically imported by the entry chunk.
Inspection shows the chunk contains **three.js core + react-dom + drei merged
together** — the `manualChunks` function in `vite.config.ts` no longer groups
correctly under Vite 8 (rolldown): the intended `vendor-react` and `vendor-r3f`
chunks are never emitted, and `vendor-three` (103 KB) holds something else.

Effect: every visitor downloads the full WebGL stack before the text-first hero
renders, even though all 3D scenes (`PhoneSwap`, `ConceptQuoteBowlCanvas`,
`StickerLayer`) are already lazy-loaded.

**Recommendation:** rework chunking for rolldown (e.g. `build.rolldownOptions`
`advancedChunks`, or drop `manualChunks` entirely and re-measure). Success
criteria: entry chunk imports no `vendor-drei`/three code; initial gzip JS drops
from ~380 KB to roughly ~100 KB, with three/drei fetched only when a 3D stage
mounts.

## 2. Security

### api/contact.ts — good overall
Honeypot field, minimum-fill-time check, per-IP rate limit, origin/referer
allowlist, payload length caps, no secrets echoed. Two small notes:

- **Subject header injection (low):** `payload.name` is interpolated into the
  email subject after only `trim()`. Strip `\r`/`\n` (or collapse whitespace)
  from `name` before building the subject. Resend's JSON API makes real header
  injection unlikely, but the fix is one line.
- The in-memory rate limiter resets per serverless instance — acceptable for a
  portfolio; just know it's best-effort.

### npm audit — 10 vulnerabilities (6 high), all transitive via @vercel/node
`undici` (multiple highs), `path-to-regexp`, `smol-toml`, etc. None of this
code runs in the deployed function — Vercel supplies its own runtime — but
`@vercel/node` sits in production `dependencies` when only its **types** are
used. **Recommendation:** move `@vercel/node` to `devDependencies` and run
`npm audit fix` for the remainder. This clears the audit noise without any
runtime change.

### Missing site-wide security headers
`vercel.json` sets headers only for `/api/*`. Add for `/(.*)`:
`X-Frame-Options: DENY` (or CSP `frame-ancestors`), `Referrer-Policy:
strict-origin-when-cross-origin`, `Permissions-Policy` (camera/mic/geolocation
off), and ideally a `Content-Security-Policy`. Note a CSP must allow
`fonts.googleapis.com` / `fonts.gstatic.com` and Vercel Analytics.

### Secrets
None committed. `.env*` ignored, `.env.example` contains placeholders only. ✅

## 3. Asset weight

- **`public/images/_archive/` (7.4 MB) is deployed but referenced nowhere** in
  code or content. Move it out of `public/` (or exclude from deploy).
- Redundant font formats: `.ttf` files (288 KB) ship alongside the `.woff2`
  versions in `public/fonts/`; only the woff2 files are needed.
- Heavy individual textures worth revisiting:
  `textures/lines-handwriting-crumpled/*_roughness.png` (3.6 MB) and
  `*_basecolor.png` (1.7 MB) — candidates for downscale/re-encode (or KTX2);
  `iphone16-pro-tex/brush_normal_{a,b}.webp` (2.5 MB each).
- Total tracked `public/` in HEAD: 29.9 MB — fine for a media-rich portfolio
  once the archive folder is removed.

## 4. Repo hygiene

- **Git pack is 205 MB** — deleted 3D sources remain in history
  (`pixel-8-pro.max` 19 MB, `.obj` files 10–20 MB, uncompressed PNGs 11–21 MB).
  Clones and CI checkouts pay this forever. Optional: rewrite history
  (`git filter-repo`) or accept the cost; either way keep future source files
  (`.max`, raw `.obj`, master PNGs) out of git as the `.gitignore` already
  does for `touch-2-master`.
- README is stale: it instructs adding `JetBrainsMono-*.woff2` to
  `public/fonts/`, but JetBrains Mono (and Lato) actually load from Google
  Fonts; only Alte Haas Grotesk is local.

## 5. Build pipeline

- `npm run build` fails after the vite step on any machine without desktop
  Chrome: `scripts/prerender.mjs` only supports `channel: 'chrome'` locally.
  **Recommendation:** honor an env override, e.g.
  `PUPPETEER_EXECUTABLE_PATH`/`PRERENDER_EXECUTABLE_PATH`, before falling back
  to the channel launch. (Works fine on Vercel/CI via `@sparticuz/chromium`.)
- Google Fonts stylesheet in `<head>` is render-blocking and a third-party
  runtime dependency. Consider self-hosting JetBrains Mono + Lato woff2 subsets
  (also simplifies a future CSP).

## 6. Dependencies (currency)

Majors available, none urgent: `@react-three/fiber` 8→9 and `react` 18→19 must
move together; `@react-three/drei` 9→10 follows fiber; `tailwindcss` 3→4 and
`typescript` 5→6 are separate migrations. Patch-level updates (`vite`,
`resend`, `sharp`, `postcss`) are safe to take anytime.

## 7. SEO & accessibility — strong

- All 9 `<img>` usages have `alt`; `prefers-reduced-motion` handled in 18
  files; `focus-visible` styling in 10 stylesheets.
- Meta/OG/Twitter tags, JSON-LD Person schema, canonical, sitemap, robots.txt,
  themed favicons, `noindex` 404 — all present and correct.
- Minor: `og:image` reuses the portrait JPEG; a dedicated 1200×630 card image
